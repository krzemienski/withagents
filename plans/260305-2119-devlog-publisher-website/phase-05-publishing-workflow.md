# Phase 5: Publishing Workflow

**Priority:** Medium
**Effort:** M (Medium)
**Status:** Pending
**Blocked By:** Phase 2, Phase 4

## Overview

Design and document the end-to-end publishing execution workflow — from finished blog post to live on the website, shared on social platforms, and tracked for performance.

## Context

- Website deployed on Vercel (Phase 2)
- 21 posts total (11 existing + 10 new from Phase 4)
- devlog-publisher skill already generates: blog_post.md, twitter_thread.md, linkedin_post.md, newsletter.md
- Stitch MCP available for last-mile design adjustments
- Analytics tracking page views, scroll depth, outbound clicks

## Requirements

### Publishing Pipeline
- Pre-publish checklist (content, visuals, SEO, companion repo)
- Automated build/deploy via Vercel on git push
- Social media content ready (X thread, LinkedIn post, newsletter)
- Post-publish monitoring (analytics, engagement)

### Scheduling
- Recommended publishing cadence (e.g., 2 posts/week)
- Optimal posting times by platform
- Cross-promotion strategy between platforms

## Implementation Steps

1. **Pre-publish checklist** (automated where possible)
   ```
   [ ] Post renders correctly in dev (`pnpm dev`)
   [ ] Frontmatter complete (title, date, description, tags, hero)
   [ ] Companion repo pushed to GitHub with README
   [ ] Hero image generated and placed in visuals/
   [ ] OG image generated (1200x630) in public/og/
   [ ] Mermaid diagrams render in dark theme
   [ ] Social cards generated (Twitter + LinkedIn)
   [ ] Social copy written (X thread, LinkedIn post, newsletter)
   [ ] Internal links working (prev/next, related posts)
   [ ] Reading time calculated
   [ ] SEO meta tags set (title, description, canonical)
   [ ] JSON-LD structured data included
   ```

2. **Publishing execution flow**
   ```
   Author writes post
     → Add to posts/{nn}-{slug}/post.md
     → Generate visuals (Stitch MCP + Mermaid)
     → Run pre-publish checklist
     → Git commit + push to main
     → Vercel auto-deploys
     → Verify live URL renders correctly
     → Post X thread (manual or scheduled)
     → Post LinkedIn article (manual or scheduled)
     → Send newsletter (if using email platform)
     → Monitor analytics for 48 hours
   ```

3. **Social media templates**
   - X/Twitter thread template (7-12 tweets, hook format)
   - LinkedIn post template (200-350 words, pattern interrupt opening)
   - Newsletter template (400-600 words, curiosity-driven subject line)
   - All generated by devlog-publisher skill, stored alongside post

4. **Post-publish monitoring**
   - Track: page views, avg read time, scroll completion rate
   - Track: outbound clicks to companion repo
   - Track: social engagement (likes, shares, comments)
   - Weekly digest of top-performing posts
   - Identify which posts drive the most repo visits

5. **Recommended publishing schedule**
   - 2 posts per week (Tue + Thu, 9am ET)
   - X thread: same day as blog, 11am ET
   - LinkedIn: next day, 8am ET
   - Newsletter: weekly digest on Friday
   - Companion repo: pushed 24h before blog (builds anticipation)

6. **Document the workflow**
   - Create `docs/publishing-workflow.md` in the blog-series project
   - Include checklist, timeline, templates, monitoring guide
   - Reference devlog-publisher skill for content generation

## Output Files

- `docs/publishing-workflow.md` — Full workflow documentation
- Post-level social content stored in `posts/{nn}-{slug}/social/`

## Success Criteria

- [ ] Pre-publish checklist documented and executable
- [ ] Publishing flow tested with at least 1 existing post
- [ ] Social media templates created for all 3 platforms
- [ ] Analytics dashboard configured for post-publish monitoring
- [ ] Publishing schedule documented with platform-specific timing
- [ ] Workflow documentation complete in `docs/`

## Risk Assessment

- **Manual overhead:** Social posting is still manual on most platforms. Mitigation: prepare all content in advance, use scheduling tools if available.
- **Analytics granularity:** Scroll depth tracking requires custom event setup. Mitigation: implement in Phase 2 analytics integration.
