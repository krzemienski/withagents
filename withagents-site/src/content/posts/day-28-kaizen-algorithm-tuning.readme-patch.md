## Featured in: Day 28 of the WithAgents launch

**Kaizen Algorithm Tuning** ships four manufacturing-discipline primitives as Claude Code skills: Five Whys, Fishbone, PDCA, and A3. They are pointed at the threshold-tuning problem that kept eating my weeks — the kind of regression where three parallel edits make a symptom disappear without anyone knowing which edit did the work.

The contract the skills enforce: one hypothesis, one change, one measurement. Two failed cycles on the same hypothesis stops the loop. The A3 artifact is the file the next session reads before touching code.

Full post on the Five Whys session that triggered the repo and the `failed-approaches.md` template: https://withagents.dev/writing/day-28-kaizen-algorithm-tuning

Pairs with: `validationforge` (what catches the symptom), `agent-constitution-framework` (what writes the rule that stops the shortcut next time).
