# LinkedIn Syndication Adapter

TypeScript module for syndicating withagents.dev blog posts to LinkedIn.

**Important:** LinkedIn's API v2 does NOT support programmatic Article publishing.
Only short UGC feed posts (shares) can be posted via API. Long-form Articles
require manual publish via the LinkedIn UI. This adapter handles both paths:

| Mode | API | Effort |
|------|-----|--------|
| Short feed post (200-350 words + link card) | Automated via `share.ts` | ~5 sec |
| Long-form Article (800-1500 words) | Manual paste via `article-prep.ts` | ~5 min |

---

## Prerequisites

### 1. LinkedIn App credentials

Create a LinkedIn app at https://developer.linkedin.com/ and enable:
- **Share on LinkedIn** product (grants `w_member_social` scope)
- **Sign In with LinkedIn using OpenID Connect** (grants `openid profile` scopes)

Add redirect URL: `http://localhost:4174/callback`

### 2. Obtain tokens (one-time per account)

```bash
export LINKEDIN_CLIENT_ID=your_client_id
export LINKEDIN_CLIENT_SECRET=your_client_secret
node scripts/linkedin-auth-local.js
```

This writes `.env.linkedin-tokens.json` at the repo root (gitignored via `.env*` pattern).

Token shape:
```json
{
  "platform": "linkedin",
  "access_token": "...",
  "refresh_token": "...",
  "author_urn": "urn:li:person:XXXXXXXX",
  "expires_in": 5183944,
  "expires_at": "2026-07-01T00:00:00.000Z",
  "fetched_at": "2026-04-19T09:00:00.000Z"
}
```

### 3. Runtime env vars

| Variable | Required | Default | Purpose |
|----------|----------|---------|---------|
| `LINKEDIN_CLIENT_ID` | Only for refresh | — | LinkedIn app client ID |
| `LINKEDIN_CLIENT_SECRET` | Only for refresh | — | LinkedIn app client secret |
| `LINKEDIN_TOKEN_FILE` | No | `.env.linkedin-tokens.json` | Override token file path |
| `SITE_BASE_URL` | No | `https://withagents.dev` | Base URL for canonical + UTM links |
| `NO_BROWSER` | No | — | Set to `1` to skip browser open in article-prep |

## Usage

All commands use `pnpm tsx` to run TypeScript directly without a build step.
Run from the repo root.

### Check / refresh tokens

```bash
pnpm tsx scripts/syndication/linkedin/oauth.ts
```

Prints `author_urn` and `expires_at`. Rotates access token via refresh_token if
expiry is within 30 minutes. Exits non-zero on auth failure.

### Post a short feed update

```bash
# Dry run (prints payload, no API call)
pnpm tsx scripts/syndication/linkedin/share.ts post-01-series-launch --dry-run

# Publish live
export LINKEDIN_CLIENT_ID=...
export LINKEDIN_CLIENT_SECRET=...
pnpm tsx scripts/syndication/linkedin/share.ts post-01-series-launch
```

**Post text source (priority order):**
1. `posts/<slug>/<slug>.linkedin-short.md` — explicit short-post sidecar (recommended)
2. Auto-generated teaser from `post.md` title + subtitle + first paragraph

**Output:**
```
[linkedin/share] Posting short update for: post-01-series-launch
  UTM URL : https://withagents.dev/posts/post-01-series-launch?utm_source=linkedin&...
  Text    : Series Launch: 23,479 AI Coding Sessions…
[linkedin/share] Published!
  URN : urn:li:ugcPost:7189562345678901234
  URL : https://www.linkedin.com/feed/update/urn:li:ugcPost:7189562345678901234/
```

### Prepare a long-form Article for manual publish

```bash
pnpm tsx scripts/syndication/linkedin/article-prep.ts post-01-series-launch
```

Opens the LinkedIn Article editor in the browser with the title pre-filled,
and writes a prep file you paste from:

```
posts/post-01-series-launch/post-01-series-launch.linkedin-article-prep.md
```

**Skip browser open:**
```bash
pnpm tsx scripts/syndication/linkedin/article-prep.ts post-01-series-launch --no-browser
# or
NO_BROWSER=1 pnpm tsx scripts/syndication/linkedin/article-prep.ts post-01-series-launch
```

**Article body source (priority order):**
1. `posts/<slug>/<slug>.linkedin.md` — explicit long-form LinkedIn sidecar (recommended for flagships)
2. `posts/<slug>/post.md` body — stripped of MDX/JSX, mermaid diagrams replaced with notes

**Manual publish steps (5 minutes):**
1. Script opens `https://www.linkedin.com/pulse/new-article/?title=<encoded-title>`
2. Copy/paste the body from the prep file into the article editor
3. Upload the cover image listed in the prep file
4. Click Publish

## Sidecar file conventions

| File | Purpose |
|------|---------|
| `posts/<slug>/<slug>.linkedin-short.md` | 200-350 word feed post text. Plain text, hashtags at bottom. |
| `posts/<slug>/<slug>.linkedin.md` | 800-1500 word LinkedIn Article body. Full Markdown. |
| `posts/<slug>/<slug>.linkedin-article-prep.md` | **Generated** by article-prep.ts. Do not hand-edit. |

Sidecar files are generated by `scripts/syndication/linkedin/` during Phase 12
execution. You can pre-write them manually to override the auto-generated content.

## Error handling

| Scenario | Behavior |
|----------|---------|
| Token file missing | Exit 1 with instructions to run auth script |
| Token expired, no refresh_token | Exit 1 with instructions to re-auth |
| Token refresh 400/401 | Exit 1 — re-auth required |
| LinkedIn API 401 | Exit 1 — suggest running oauth.ts to verify |
| LinkedIn API 422 | Exit 1 — post text too long or invalid URL |
| LinkedIn API 5xx | Exit 1 with status and body |
| Post directory not found | Exit 1 with expected path |
| Browser open failure | Warning only, continues (non-fatal) |

## Module structure

```
scripts/syndication/linkedin/
├── types.ts          — shared TypeScript interfaces (token, share, article)
├── oauth.ts          — token load + refresh-token rotation
├── share.ts          — short feed post via UGC Posts API
├── article-prep.ts   — long-form article prep + browser open
└── README.md         — this file
```

**Import pattern** (for future scheduler/webhook integration):
```typescript
import { loadTokens } from "./linkedin/oauth.js";
import type { LinkedInShareRequest } from "./linkedin/types.js";
```
